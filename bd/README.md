# SysEye Backend

Бэкенд-сервис для системы агентов диагностики и управления командами.

## Документация, запуск и общая готовность

- Подробный запуск: [run.md](/home/user/Desktop/syseye/bd/run.md)
- Конфигурация backend: [configuration.md](/home/user/Desktop/syseye/bd/configuration.md)

## Общая готовность решения

Backend покрывает основные контуры системы:

- управление агентами
- запуск команд и сценариев на агентах
- realtime-обновления статусов и логов
- аутентификацию пользователей
- подтверждение email и сброс пароля
- экспорт логов и диагностики
- сбор телеметрии и логов
- фоновые задачи и уведомления

Сервис рассчитан на запуск в Docker-окружении с PostgreSQL, Redis и внешними системами мониторинга.

## Архитектура

Сервис работает как центральная backend-точка для пользовательского интерфейса и агентов.

Основные потоки:

1. frontend обращается в backend по HTTP API
2. frontend получает realtime-события через SignalR
3. агент подключается к backend и получает задания
4. агент отправляет статусы выполнения и chunk-логи обратно
5. backend сохраняет выполнение, публикует события и экспортирует логи
6. фоновые задачи выполняются через Hangfire

## Основные возможности

### Агенты

- регистрация и переподключение агентов
- heartbeat и отслеживание online/offline
- привязка к пользователю
- запуск команд и сценариев
- отмена выполняемых задач

### Команды и сценарии

- сохранённые команды для Linux и Windows
- системные диагностические команды
- пользовательские команды
- сценарии из нескольких шагов
- запуск на одном агенте и по группе

### История выполнения

- статусы `queued`, `running`, `success`, `error`, `cancelled`, `interrupted`
- хранение stdout/stderr
- realtime-обновление карточек выполнения
- просмотр execution logs
- regex-логика для выборки значимых фрагментов

### Аналитика

- сводка по агентам
- рейтинг агентов
- метрики по запускам, ошибкам и среднему времени
- активность по дням

### Экспорт

Поддерживается экспорт выполненных задач и аналитики:

- `Json` — полный экспорт в формате JSON
- `Csv` — табличный экспорт
- `Pdf` — отчёт с аналитикой и таблицами

Если фильтр не передан, экспорт должен работать по всем данным пользователя.

## 🔐 Аутентификация

Используется JWT-модель:

- `Access Token`
- `Refresh Token`
- хранение в `Cookies`

Дополнительно поддерживается:

- подтверждение email
- сброс пароля через email
- серверные пользовательские сессии

## 📬 Email система

Схема работы:

1. API ставит задачу в очередь
2. Hangfire обрабатывает её
3. письмо отправляется в фоне через SMTP

Используется для:

- подтверждения email
- сброса пароля
- сервисных уведомлений

## 📡 Логирование и мониторинг

Пайплайн логов:

1. агенты отправляют логи в backend по websocket / SignalR
2. backend публикует их как execution logs
3. backend экспортирует telemetry в OpenTelemetry Collector по gRPC
4. collector обрабатывает и отправляет данные в Loki

Также поддерживаются:

- файловые логи backend
- консольные логи
- запросы логов выполнения из Loki

## Важные замечания

- Конфигурация валидируется на старте через `IOptions` и `IValidateOptions`
- Секреты лучше передавать через Docker Secrets, а не хранить в `.env`
- Для production важны корректные `AllowedOrigins`, SMTP-настройки и JWT secret
- Если backend запускается из старого Docker image, изменения в исходниках не применятся без новой сборки
